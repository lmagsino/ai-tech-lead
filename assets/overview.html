Your AI Technical Co-Founder

AI Tech Lead

Founders get a CTO who never sleeps. Developers get a senior tech lead who challenges, reviews, and ships with you.

Specs before code Evidence before conclusions Behavior tests before implementation Blast radius before committing Patterns over incidents
Modes
🔭
/strategy
"I have an idea"
Validates your idea — market, competition, tech feasibility, business model. Tough YC partner energy.
→ STRATEGY.md with GO or STOP
🎨
/design
"Design the product"
User journeys, AI interactions, screen-by-screen specs. Thinks in flows, not features.
→ DESIGN.md
🤔
/challenge
"Should we build this?"
Challenges whether to build something. Always asks if there's an AI-native alternative.
→ GO / RETHINK / KILL with reasoning
📐
/blueprint
"Write a spec"
Probes for gaps, designs AI components, defines acceptance criteria and cost estimates.
→ specs/[feature].md
⚙️
/forge
"Build this"
AI infrastructure first, tests first, clean code gate before every commit. Builds exactly what the spec says.
→ Committed code with passing tests
🛡️
/guard
"Review this code"
5-pass review: structure, smells, security, clean code, AI components. CRITICAL/HIGH block merge.
→ Severity-ranked scorecard
🔬
/hunt
"There's a bug"
Classifies first, traces the causal chain, fixes with regression test and RCA document.
→ Fix + docs/rca/[date]-[description].md
🚀
/launch
"Ship it"
Pre-launch checklist: functionality, AI systems, security, infrastructure. No soft approvals.
→ GO or NO-GO
🗺️
/roadmap
"What do we build first?"
Prioritizes features, sequences the MVP, build vs buy decisions, developer handoff brief.
→ ROADMAP.md
