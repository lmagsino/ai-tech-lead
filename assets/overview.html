Engineering Intelligence

ATHENA

Spec-driven workflows for AI coding agents. Nine skills that guide you from idea to production — with discipline.

Specs before code Evidence before conclusions Behavior tests before implementation Blast radius before committing Patterns over incidents
Nine Skills
🔭
/athena scope
"Should we build this?"
Product, design, and engineering challenge before a line of code is written.
→ GO / RETHINK / KILL with reasoning
📐
/athena spec
"Write a spec"
Probe for edge cases, acceptance criteria, and out-of-scope explicitly.
→ specs/[feature].md
⚙️
/athena build
"Build this"
Test-first implementation against the spec. Clean code gate before each commit.
→ Implementation + behavior tests
🔬
/athena debug
"There's a bug"
Trace to root cause. Scoped fix. Regression test. RCA document.
→ docs/rca/[date]-[description].md
🛡️
/athena review
"Review this code"
5-pass scorecard: SOLID, smells, anti-patterns, security, spec conformance.
→ Severity-ranked findings with file:line
🖥️
/athena qa
"Check the UI"
Multi-viewport screenshots, console scan, a11y check, spec UI criteria.
→ Visual health report + screenshots
🔄
/athena refactor
"Refactor this"
Behavior tests first. Incremental migration, one pattern at a time. Never rewrites.
→ Modernized code + parity report
⚡
/athena fix
"Quick fix"
Trivial changes under 3 files. Lightweight gate. No ceremony.
→ Fix committed
📊
/athena retro
"What did we learn?"
Analyze git history. Update lessons.md. Tech debt roadmap by ROI.
→ lessons.md + debt roadmap
